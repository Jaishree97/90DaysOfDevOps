# Day 08 - Cloud Deployment & Log Management

## Objective

Deploy a web server on an AWS EC2 instance, verify the service status, collect logs, and download log files to the local machine.

---

# Step 1: Connect to EC2 Instance

### Command

```bash
ssh -i "miya.pem" ubuntu@ec2-34-221-88-167.us-west-2.compute.amazonaws.com
```

### Why I Used It

This command securely connects my local machine to the Ubuntu EC2 instance using SSH. The `-i` option specifies the private key required for authentication.

---

# Step 2: Update Package Repository

### Command

```bash
sudo apt update -y
```

### Why I Used It

This command refreshes the package index from Ubuntu repositories and ensures the latest package information is available before installing software.

---

# Step 3: Install Nginx

### Command

```bash
sudo apt install nginx -y
```

### Why I Used It

This command installs the Nginx web server on the EC2 instance. The `-y` option automatically approves the installation prompts.

---

# Step 4: Verify Nginx Service

### Command

```bash
systemctl status nginx
```

### Why I Used It

This command checks whether the Nginx service is active and running correctly after installation.

---

# Step 5: Verify Installed Nginx Package

### Command

```bash
dpkg -l | grep nginx
```

### Why I Used It

This command lists all installed Nginx-related packages and confirms that Nginx has been installed successfully.

---

### Command

```bash
dpkg -s nginx
```

### Why I Used It

This command displays detailed information about the installed Nginx package, including version, status, dependencies, and description.

---

# Step 6: Create a Custom Web Page

### Command

```bash
cd /var/www/html
```

### Why I Used It

This command navigates to the default Nginx web root directory where website files are stored.

---

### Command

```bash
echo "Jaishree Chaure, Nagpur" | sudo tee index.nginx-debian.html
```

### Why I Used It

This command replaces the default Nginx welcome page with my custom webpage content.

---

# Step 7: View Running Services

### Command

```bash
systemctl list-units --type=service --state=running
```

### Why I Used It

This command displays all currently running system services and helped me verify that Nginx was running.

---

# Step 8: View Nginx Logs

### Command

```bash
sudo journalctl -u nginx -n 20
```

### Why I Used It

This command displays the last 20 log entries generated by the Nginx service. It helps in monitoring and troubleshooting service activity.

---

# Step 9: Save Logs to a File

### Command

```bash
sudo journalctl -u nginx -n 20 > ~/nginx-logs.txt
```

### Why I Used It

This command saves the latest Nginx logs into a file named `nginx-logs.txt` in my home directory.

---

# Step 10: Verify Log File

### Command

```bash
ls -l
```

### Why I Used It

This command verifies that the log file was created successfully and displays its permissions and size.

---

### Command

```bash
cat nginx-logs.txt
```

### Why I Used It

This command displays the contents of the saved log file for verification.

---

# Step 11: Download Log File to Local Machine

### Command

```bash
scp -i "miya.pem" ubuntu@ec2-34-221-88-167.us-west-2.compute.amazonaws.com:~/nginx-logs.txt .
```

### Why I Used It

This command securely downloads the log file from the EC2 instance to my local machine. The `.` at the end specifies the current local directory as the download destination.

---

```bash
ssh -i "miya.pem" ubuntu@ec2-34-221-88-167.us-west-2.compute.amazonaws.com

```

# Challenges Faced

### 1. SCP Download Failed Initially

**Issue:**

The SCP command initially failed with:

```text
Identity file miya.pem not accessible
```

**Solution:**

I realized that the PEM file was not present in my current local directory. After navigating to the Downloads folder where the PEM file was stored, the SCP command worked successfully.

---

### 2. Creating Custom Web Page

**Issue:**

Initially, I attempted to save the HTML file in an incorrect path and received a "No such file or directory" error.

**Solution:**

I navigated to the correct Nginx web root directory (`/var/www/html`) and recreated the file successfully.
