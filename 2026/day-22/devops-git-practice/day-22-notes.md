# Day 22 Notes

## 1. Difference between git add and git commit

git add moves changes to the staging area.

git commit permanently saves staged changes to repository history.

---

## 2. What does staging area do?

The staging area allows us to review changes before committing them.

---

## 3. What does git log show?

It shows commit history including commit ID, author, date and message.

---

## 4. What is .git folder?

The .git folder stores repository history, commits, branches and configuration.

If deleted, Git tracking is lost.

---

## 5. Difference between Working Directory, Staging Area and Repository

Working Directory:
Files currently being edited.

Staging Area:
Prepared changes waiting for commit.

Repository:
Committed history stored by Git.
